================================================================================
                              HOW IT WORKS
================================================================================

CREATION FLOW:
1. User joins configured "creator" voice channel
2. Bot checks: not banned, not on cooldown, not already has channel, guild under limit
3. Bot creates new voice channel in configured category
4. User auto-moved to new channel
5. Channel record stored in database
6. Cooldown applied, inactivity timer started

DELETION FLOW:
1. When last user leaves → 60-second timer starts
2. If someone joins within 60 seconds → timer cancelled, activity resets
3. If 60 seconds pass with no joins → channel deleted automatically
4. Optional: if no voice activity for X minutes → auto-deleted

LOCKING SYSTEM:
1. Owner can lock channel → only owner + permitted users can join
2. Owner can permit users → explicitly allow them in locked channel
3. Owner can deny users → prevent them from joining
4. Permissions tracked per-channel in database

CLEANUP PROCESS:
- Runs every 60 seconds automatically
- Checks all temp VC records against actual Discord channels
- Removes orphaned database records
- Deletes empty channels with no scheduled deletion
- Handles all errors gracefully

================================================================================
                            QUICK START
================================================================================

1. Copy TempVoice folder to /Modules/

2. Import and load in bot:
   import tempVoiceModule from './Modules/TempVoice';
   const commands = tempVoiceModule.loadCommands();

3. Initialize in ready event:
   tempVoiceModule.initializeModule(client);

4. Create two Discord channels:
   - Creator voice channel
   - Category for temp VCs

5. Run setup commands:
   /vcconfig creator <channel>
   /vcconfig category <category>

6. Users can now use /vc to create channels!

See QUICKSTART.md for detailed instructions.
