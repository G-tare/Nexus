MODULE OVERVIEW
================================================================================

The Sticky Messages module keeps important messages "stuck" at the bottom of
Discord channels by automatically deleting and resending them. It supports
intelligent activity-based detection that adapts to channel traffic levels.

Key Capabilities:
  - Automatic message re-posting at fixed or activity-based intervals
  - Intelligent activity detection (low/medium/high)
  - Multiple sticky messages per channel with priority ordering
  - Support for plain text and Discord embeds
  - Edit sticky content without recreating
  - Rate limiting to prevent spam
  - Comprehensive permission and error handling
  - Full audit logging support

CORE FEATURES & SPECIFICATIONS
================================================================================

Feature 1: Activity Detection
──────────────────────────────
  Implementation: ActivityTracker class with rolling 1-minute window
  Thresholds:
    - Low activity:    <1 message/minute  → 3 message threshold
    - Medium activity: 1-5 messages/minute → 8 message threshold
    - High activity:   >5 messages/minute → 15 message threshold
  
  Benefits:
    - Adapts to channel traffic automatically
    - Prevents spam in busy channels
    - Maintains visibility in quiet channels

Feature 2: Three Operation Modes
─────────────────────────────────
  
  1. Interval Mode
     └─ Re-stick after fixed number of messages
     └─ Configurable per sticky (1-100)
     └─ Default: 5 messages
     └─ Best for: Consistent, predictable channels
  
  2. Activity Mode
     └─ Dynamic thresholds based on real-time activity
     └─ Uses: 3/8/15 based on activity level
     └─ Best for: Mixed-activity channels
  
  3. Hybrid Mode
     └─ Uses configured interval as minimum
     └─ Scales up based on activity level
     └─ Best for: All-purpose, adaptive usage

Feature 3: Multiple Stickies Per Channel
─────────────────────────────────────────
  - Up to 3 per channel by default (configurable: 1-10)
  - Priority-based ordering (0-999)
  - Each with independent interval
  - Each tracked separately in database
  - Queued for re-sticking in priority order

Feature 4: Content Management
──────────────────────────────
  - Plain text message support
  - Full Discord embed support (via APIEmbed)
  - Edit without recreating
  - Automatic resend on Discord deletion
  - Graceful handling when message deleted externally

Feature 5: Rate Limiting
────────────────────────
  - 5 second minimum between re-sticks per message
  - Per-sticky tracking with timestamp map
  - Prevents API rate limiting
  - Prevents spam in chat

Feature 6: Error Handling & Resilience
────────────────────────────────────────
  - Permission validation before operations
  - Auto-deactivates on permission loss
  - Handles deleted channels gracefully
  - Recovers from deleted messages
  - Comprehensive error logging

Feature 7: Audit Logging
────────────────────────
  Events emitted:
  - STICKY_CREATED: New sticky message
  - STICKY_REMOVED: Sticky deleted
  - STICKY_EDITED: Sticky modified (with change details)
  - STICKY_CONFIG_UPDATED: Configuration changed

Feature 8: Configuration Management
──────────────────────────────────────
  Guild-level settings:
  - enabled: Enable/disable module (default: true)
  - mode: 'interval' | 'activity' | 'hybrid' (default: 'interval')
  - maxStickiesPerChannel: 1-10 (default: 3)
  - deleteBotMessage: Boolean (default: true)

================================================================================
OPERATION FLOW
================================================================================

Message Creation Flow
──────────────────────

1. User sends message in channel with sticky(ies)
   ↓
2. messageCreate event fires
   ↓
3. Record message in ActivityTracker
   ↓
4. Get all active stickies for channel
   ↓
5. For each sticky:
   a. Increment messagesSince counter
   b. Calculate threshold (based on mode + activity)
   c. Check if messagesSince >= threshold
   d. Check rate limit (5 sec minimum)
   e. If all checks pass:
      - Delete old message (if enabled)
      - Send new message to channel
      - Update currentMessageId in DB
      - Reset messagesSince counter

Re-sticking Logic
──────────────────

Interval Mode:
  threshold = sticky.interval
  if (messagesSince >= threshold) → re-stick

Activity Mode:
  activityLevel = getActivityLevel(channelId)
  threshold = {low: 3, medium: 8, high: 15}[activityLevel]
  if (messagesSince >= threshold) → re-stick

Hybrid Mode:
  activityLevel = getActivityLevel(channelId)
  activityThreshold = {low: 3, medium: 8, high: 15}[activityLevel]
  threshold = max(sticky.interval, activityThreshold)
  if (messagesSince >= threshold) → re-stick

CONFIGURATION EXAMPLES
================================================================================

Example 1: Fixed Interval Mode (Default)
─────────────────────────────────────────
  /stickyconfig mode interval
  /stick channel:#announcements content:"📢 Important announcement!" interval:8

  Result: Sticks every 8 messages, regardless of activity

Example 2: Activity-Adaptive Mode
──────────────────────────────────
  /stickyconfig mode activity
  /stick channel:#general content:"Welcome!" interval:5

  Result: 
    - Quiet channel: sticks every 3 messages
    - Normal channel: sticks every 8 messages
    - Busy channel: sticks every 15 messages

Example 3: Hybrid Mode (Recommended)
─────────────────────────────────────
  /stickyconfig mode hybrid
  /stick channel:#rules content:"Rules here..." interval:10

  Result:
    - Minimum 10 messages before re-sticking
    - But scales up if activity is high
    - Prevents disappearing in busy channels
    - Maintains reasonable frequency in quiet channels

Example 4: Multiple Priority-Based Stickies
────────────────────────────────────────────
  /stick channel:#rules content:"Rule 1" priority:100 interval:15
  /stick channel:#rules content:"Rule 2" priority:90 interval:15
  /stick channel:#rules content:"Rule 3" priority:80 interval:15

  Result: All three messages stick, highest priority reappears last

================================================================================
ERROR HANDLING MATRIX
================================================================================

Scenario 1: Missing Permissions
  Condition: Bot lacks SendMessages or ManageMessages
  Detection: Checked before operations
  Action: Prevent operation, return error message
  Recovery: Auto-deactivate on consecutive failures

Scenario 2: Deleted Message
  Condition: Old sticky message deleted externally
  Detection: messageDelete event or fetch fails
  Action: Clear currentMessageId, resend immediately
  Recovery: Automatic

Scenario 3: Deleted Channel
  Condition: Channel containing sticky deleted
  Detection: channelDelete event
  Action: Deactivate all stickies in channel
  Recovery: Manual recreation by staff

Scenario 4: Permission Changes
  Condition: Bot permissions revoked mid-operation
  Detection: Send/edit/delete operation fails
  Action: Log error, deactivate sticky
  Recovery: Manual re-activation with /stick after fix

Scenario 5: Database Error
  Condition: DB query fails
  Detection: Query exception
  Action: Log error, return to user
  Recovery: Retry on next message
